# Skill: i18n — Afegir text nou

## Regla

Tot text nou d'interfície usa `tr("clau.subclau")` amb claus definides als fitxers JSON:

- `src/i18n/locales/ca.json`
- `src/i18n/locales/es.json`
- `src/i18n/locales/fr.json`
- `src/i18n/locales/pt.json`

---

## Checklist per afegir claus noves

1. **Definir clau:** triar un namespace coherent (ex: `donors.loadMore`).
2. **Afegir a 4 idiomes:** CA, ES, FR, PT — tots obligatoris.
3. **Validar:** `npm run build` ha de passar sense errors.
4. **Commit:** incloure els 4 fitxers JSON al commit.

---

## Prohibicions

- **Hardcode de strings UI** — mai posar text directament al JSX/TSX.
- **`t("...")` per text nou** — `t("...")` NO existeix al sistema i18n. Usar `tr("...")`.
- **Legacy `t.xxx.yyy`** — només tocar si és text antic existent que s'està editant. Text nou sempre amb `tr()`.
