## Mètode de revisió contra codi

Cada bloc funcional s'ha de revisar amb el mateix criteri:

| Estat | Significat |
|-------|------------|
| ✅ Validada | La funcionalitat està implementada i hi ha evidència clara al codi. |
| 🟡 Parcial / risc | La funcionalitat existeix, però té algun límit, dependència o risc que cal controlar. |
| ⚠️ Documentada, pendent de contrast | La funcionalitat està descrita, però encara no s'ha revisat prou contra el codi. |
| ❌ No trobada / obsoleta | No s'ha trobat implementació vigent o el codi apunta a funcionalitat retirada. |

Per donar una funcionalitat per validada cal deixar constància de:

- pantalla o flux principal
- dades que llegeix o escriu
- serveis, helpers o API implicats
- proves automàtiques existents o prova manual mínima
- riscos operatius o fiscals si n'hi ha
