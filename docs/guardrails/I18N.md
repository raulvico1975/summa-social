# Skill: i18n - afegir o editar textos

## Regla curta

El sistema actual conviu amb dos formats vàlids:

- `t.xxx.yyy` -> sistema legacy tipat en `src/i18n/*.ts`
- `tr('clau.subclau')` -> sistema nou amb JSON a `src/i18n/locales/*.json`

L'únic patró prohibit és `t("...")`: aquesta funció no existeix.

## Criteri d'ús

- Text nou: preferir `tr('clau.subclau')`
- Codi legacy que ja viu en `t.xxx.yyy`: pots mantenir-lo així si el canvi és local i no compensa migrar el bloc
- Si migres un bloc a `tr()`, afegeix les claus JSON corresponents

Nota important:

- `pt` només existeix al sistema JSON; per això el text nou ha d'anar preferentment a `tr()`

## Checklist per text nou amb `tr()`

1. Triar namespace coherent
2. Afegir la clau a `src/i18n/locales/ca.json`
3. Replicar-la a `src/i18n/locales/es.json`, `src/i18n/locales/fr.json` i `src/i18n/locales/pt.json`
4. Validar:

```bash
npm run i18n:check
npm run i18n:check-tr-keys
```

## Guies i ajuda: excepció important

No editis manualment claus `guides.*` dins `src/i18n/locales/*.json` si el canvi ve de la capa Help/Guides.

En aquest cas, la font bona és:

- `help/topics/*.ca.md`
- `help/topics/*.es.md`

I després cal regenerar:

```bash
npm run help:build-guides-adapter
```

## Prohibicions

- Hardcode de strings UI al JSX/TSX quan el text és estable d'interfície
- `t("...")`
- Afegir només CA i deixar ES/FR/PT per després
