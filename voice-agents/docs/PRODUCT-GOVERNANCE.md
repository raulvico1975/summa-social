# Product Governance

## Principi rector

No desenvolupem mes canal del necessari. En aquesta entrega nomes validem si el `web-agent` text-first aporta valor real a Summa.

## Regla de canal

- `text-first` per captacio i diagnostic web
- fora d'abast: veu, Live, demo guiada, suport operatiu
- si text no supera un formulari o un xat generic, la fase no passa el tall

## Agent obligatori

El `product-director-agent` es obligatori a totes les fases.

### Abans de fase

- valida el problema real
- fixa per que el canal triat es la millor via
- defineix evidencia minima i scope

### Durant la fase

- revisa desviacions de scope
- retalla feina sense aportacio clara
- protegeix el focus de producte

### Tancament de fase

- revisa evidencia
- emet `GO`, `GO WITH TRIM`, `HOLD` o `NO-GO`
- documenta la decisio i el seguent pas

## Criteri de decisio

Cada fase s'avalua amb aquestes preguntes:

1. Quin problema concret resolem?
2. Per que el text-first aporta menys friccio en aquest cas?
3. Que ha de notar una persona usuaria o comercial respecte a un xat generic?
4. Quin risc afegim?
5. Quina evidencia minima ho demostrara?

## Definicio de valor

Una funcionalitat nomes compta com a valor si compleix almenys una d'aquestes:

- redueix friccio d'una demo o primera conversa comercial
- converteix millor que una UI mes simple en el mateix context d'us
- deixa una conclusio d'encaix i un seguent pas clar
