# Summa Social HyperFrames Design Contract

## Role

This file governs future HyperFrames authoring for premium Summa product film.
HyperFrames is a runtime and composition tool, not the upstream proof contract.

## Identity to preserve

- calm
- exact
- premium without spectacle
- product-led
- operationally credible

## Preferred visual behavior

- real UI dominates the frame
- product footage should feel full-bleed or nearly full-bleed
- motion exists to guide the eye, not to announce itself
- typography should be sparse and deliberate
- blue remains semantic and restrained

## What to avoid

- framed product cards as the default layout
- poster headlines sitting above an embedded app window
- lower-third systems as a house style
- caption rails as a house style
- decorative gradients, shadows, glows, and flourishes used to fake quality
- fixed 4-scene marketing structure

## Composition law

- build from proof beats, not marketing scenes
- one job per beat
- one proof target per beat
- one movement intention per beat
- if the UI already explains the moment, remove copy

## Runtime note

The runtime may keep sandbox compositions for technical validation.
Sandbox pieces are not visual precedent unless explicitly approved.
