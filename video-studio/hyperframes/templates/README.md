# Summa Social HyperFrames - Template Registry

## Purpose
This folder stores the master explainer template and any future derived templates for the canonical `functional-explainer` path.

## Canonical template path
- `functional-explainer-master-16x9.md` is the master reference for 16:9 functional explainers.
- Future templates must inherit the same product-led structure unless the brand contract changes.
- All approved templates must be built from real captures and production briefs.

## Registry rules
- Keep master templates explicit about inputs, outputs, and reuse constraints.
- Keep research-only material out of this registry.
- Do not promote `05` into a template precedent.
- A template may be reused only if it still reflects the current canonical explainer system.

## Relationship to the rest of the studio
- Briefs define the objective.
- Captures define the proof.
- Shots define the reusable scene logic.
- Templates define how those parts become a producible explainer.
